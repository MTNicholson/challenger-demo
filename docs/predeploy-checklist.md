# Predeploy-чек-лист

## Перед публикацией

- [ ] `git status` не показывает незапланированных или незакоммиченных изменений.
- [ ] `npm run build` проходит успешно.
- [ ] `npm run lint` проходит успешно.
- [ ] `npm run check` проходит успешно.
- [ ] Вручную пройдены пользовательский, брендовый и сканерный демо-сценарии из `docs/qa-checklist.md`.
- [ ] Проверены 404 и основные ширины экранов.

## Предположения окружения

- Node.js и npm совместимы с версиями, ожидаемыми текущими `next` и `package-lock.json`.
- Установка выполняется через `npm ci`, запуск production-сборки — через `npm run build` и `npm run start`.
- Для работы прототипа не требуются переменные окружения или внешние секреты.
- В проекте намеренно нет реальных backend, авторизации, базы данных и API routes.
- Интерфейс использует только моковые данные из `src/data`; камера и внешние интеграции не требуются.

## Рекомендуемая публикация

Для следующего демо рекомендуется Vercel как наиболее прямой deployment target для существующего Next.js-приложения. До публикации следует сохранить текущий lock-файл, выбрать поддерживаемую версию Node.js и выполнить полный чек-лист выше.
