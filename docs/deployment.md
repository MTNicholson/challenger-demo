# Подготовка к развёртыванию

Проект остаётся frontend-only прототипом на моковых данных. Ему не нужны база данных, секреты, backend или внешние сервисы.

## Production-команды

Сборка:

```bash
npm run build
```

Запуск собранного приложения:

```bash
npm run start
```

По умолчанию Node.js-сервер слушает порт `3000`. При необходимости порт можно передать стандартным способом, поддерживаемым выбранным хостингом.

## Проверки перед развёртыванием

Рекомендуемый порядок:

```bash
npm install
npm run check
```

После автоматических проверок вручную пройти пользовательский, брендовый и сканерный сценарии, открыть неизвестный маршрут для проверки 404 и свериться с [`predeploy-checklist.md`](predeploy-checklist.md).

## Вариант A: Vercel или Next.js-хостинг

1. Подключить существующий репозиторий к хостингу с поддержкой Next.js.
2. Использовать `npm install` как команду установки и `npm run build` как команду сборки.
3. Не добавлять переменные окружения: текущему прототипу они не нужны.
4. После публикации вручную проверить основные демо-маршруты.

Vercel является самым прямым вариантом для демо Next.js. Фактическое внешнее развёртывание в рамках этого чекпоинта не выполняется.

## Вариант B: небольшой VPS с Node.js

На сервере нужны Node.js LTS, npm и исходный код проекта. После установки зависимостей выполнить проверку, сборку и `npm run start`. Для постоянной работы позднее можно добавить менеджер процессов и reverse proxy, но текущему демо они не обязательны. Практические шаги перечислены в [`vps-deploy-notes.md`](vps-deploy-notes.md).

## Windows: локальная разработка

- Команды проекта одинаково запускаются в PowerShell, Windows Terminal и терминале редактора.
- Убедиться, что Node.js и npm доступны в `PATH`.
- Для локальной разработки использовать `npm run dev`; production-режим требует сначала выполнить `npm run build`.
- Если порт `3000` занят, Next.js предложит другой порт для dev-сервера; production-порт следует настроить явно в окружении запуска.

## Ограничения прототипа

Это демонстрационный frontend, а не production-сервис: нет авторизации, backend, базы данных, реального QR/сканирования камерой, платежей и внешних интеграций. Все продуктовые данные берутся из `src/data`.
