# Заметки по запуску на VPS

Высокоуровневый сценарий для небольшого Linux VPS:

1. Установить актуальную LTS-версию Node.js и npm.
2. Клонировать существующий репозиторий и перейти в каталог проекта.
3. Установить зависимости:

   ```bash
   npm install
   ```

4. Запустить полную проверку:

   ```bash
   npm run check
   ```

5. Если сборка не была сохранена после проверки, собрать приложение:

   ```bash
   npm run build
   ```

6. Запустить production-сервер:

   ```bash
   npm run start
   ```

`npm run check` уже включает production-сборку, поэтому отдельный повтор `npm run build` обычно не нужен. Он оставлен в последовательности как явный шаг для случаев, когда артефакт сборки был удалён или код изменился после проверки.

Позднее перед Node.js-сервером можно поставить Nginx или другой reverse proxy, добавить HTTPS и менеджер процессов. Для текущего frontend-only демо это не является обязательной частью чекпоинта. База данных, миграции, секреты и переменные окружения сейчас не требуются.
