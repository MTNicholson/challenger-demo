# QA-чек-лист

## Автоматические проверки

- [ ] `npm run lint` завершается без ошибок.
- [ ] `npm run build` завершается без ошибок и перечисляет ожидаемые маршруты.
- [ ] `npm run check` последовательно выполняет lint и build.

## Ручная проверка сценариев

- [ ] Пользовательский поток кликается целиком: `/` → `/app` → `/app/challenges` → `/app/challenges/coffee-route` → `/app/active-challenge` → `/app/reward`.
- [ ] Поток бренда кликается целиком: `/` → `/brand` → `/brand/challenges` → `/brand/challenges/new` → `/brand/preview` → `/brand/analytics` → `/brand/rewards` → `/brand/scanner` → `/brand/scan-result`.
- [ ] Быстрый поток сканера кликается целиком: `/` → `/brand/scanner` → `/brand/scan-result`.
- [ ] Нижняя навигация пользовательского приложения ведёт на главную, челленджи, карту и профиль.
- [ ] Боковая навигация кабинета бренда ведёт на все разделы и корректно отмечает активный экран.

## 404 и ссылки

- [ ] Неизвестный адрес, например `/not-a-real-page`, показывает русскоязычную страницу 404.
- [ ] Неизвестный челлендж, например `/app/challenges/not-a-real-id`, показывает страницу 404.
- [ ] С 404 работают переходы на `/`, `/app` и `/brand`.
- [ ] В основных сценариях нет `#`-заглушек; якорные ссылки лендинга ведут к существующим секциям.

## Адаптивность

- [ ] Лендинг не создаёт горизонтальную прокрутку на ширинах 320, 375, 768 и 1440 px.
- [ ] Пользовательское приложение остаётся центрированным и не шире 480 px.
- [ ] Нижняя навигация не перекрывает последние основные действия и контент.
- [ ] Карточки и заголовки не выходят за границы на ширинах 320 и 375 px.
- [ ] Кабинет бренда остаётся удобным на desktop-ширинах 1024 и 1440 px.

## Известные ограничения

- Прототип не использует backend, авторизацию, базу данных и API routes.
- Все данные берутся из `src/data` и сбрасываются при обновлении страницы.
- Сканер не обращается к камере: код и результат проверки моковые.
- Поля форм не сохраняют изменения и не отправляют данные.
- Карта, QR-коды, аналитика и награды не подключены к реальным сервисам.
